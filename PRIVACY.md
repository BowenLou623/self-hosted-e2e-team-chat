# Privacy

The official open-source project itself does not collect user chat content.

Self-hosted instance data is controlled by the person or organization that deploys it. Local databases and runtime folders may contain contacts, messages, device information, keys, metadata, configuration, and logs.

The maintainers do not provide a public hosted messaging service and do not store user chat content for public users.

If a deployer provides access to other people, the deployer should prepare their own privacy notice, consent process, retention policy, deletion process, and compliance review.

Recommendations:

- Keep logs minimal.
- Avoid public internet exposure unless you understand the risks.
- Do not upload real chat data to GitHub issues.
- Do not submit logs containing personal information, private keys, access tokens, addresses, or private messages.
- Remove or redact local runtime data before sharing bug reports.
